# DevLog LLM Template: Formal

Use this template for professional work logs, timesheets, status updates, or contexts where polished wording is preferred.

## Intent

Produce precise, professional entries that can be shared with teammates, managers, clients, or pasted into a formal work summary.

## Inputs To Consider

- Deliverable or work area.
- Problem resolved or investigation performed.
- Review, implementation, or documentation status.
- Verification state and known limitations.

## Output Guidance

- Write one complete sentence per entry.
- Use polished, professional wording without sounding inflated.
- Prefer accurate verbs such as `Implemented`, `Resolved`, `Investigated`, `Validated`, `Documented`, `Reviewed`, `Refined`, or `Prepared`.
- State incomplete or unverified work clearly and professionally.
- Avoid casual phrasing and avoid exaggerating impact.

## Entry Pattern

```text
<Professional past-tense verb phrase> <specific deliverable/work area>, <context or purpose>, <verification/status when relevant>.
```

## Examples

```text
Resolved the empty summary display issue by updating frontmatter parsing in the summary loader; automated tests have not yet been run.
```

```text
Reviewed the checkout API changes and documented implementation risks and follow-up recommendations without modifying code.
```

```text
Prepared DevLog CLI style templates for future LLM integration and harness-based entry generation.
```

## Avoid

```text
Got the summary thing working
Poked at the loader bug
Wired up some templates
```
